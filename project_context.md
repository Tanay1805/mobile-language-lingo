# LanguageLearnerApplication - Project Context

## Overview
**LanguageLearnerApplication** is a dynamic, multi-modal language learning platform built with a Flutter frontend, a Node.js/Express backend, and powered by Supabase for database and authentication. The project utilizes Google's Gemini AI to dynamically generate quizzes and flashcards based on interactive content, specifically designed around a Netflix-inspired learning framework.

## Tech Stack
- **Frontend**: Flutter (Dart)
  - UI strictly follows a premium, responsive, and modern design system (Netflix aesthetic).
  - Uses `supabase_flutter` for authenticated actions.
  - Key plugins: `url_launcher` (for Calendly), `image_picker`, `google_fonts`, `fl_chart`.
- **Backend**: Node.js & Express
  - Integrates with the `@google/generative-ai` SDK (Gemini API).
  - Serves custom generation endpoints for Quizzes and Flashcards.
- **Database & Auth**: Supabase
  - Relational PostgreSQL database.
  - Handles User Auth and Secure Storage Buckets (e.g., `avatars`).

## Core Features Implemented

### 1. Dashboard & Navigation
- A unified Dashboard serving as the central hub.
- Tracks `Progress Statistics` and `Activity Charts` representing weekly learning data.
- Top navigation bar for quick access between "Content Courses" (Netflix grid) and "Schedule" (Calendly integration).

### 2. Authentication Flow & Profiles
- Sign-up flow includes direct image uploads to a public Supabase Storage bucket.
- A `profiles` table automatically syncs with `auth.users`, storing `username` and `avatar_url`.
- Dashboards load the avatar dynamically via direct DB queries to bypass cached auth tokens.

### 3. AI-Powered Learning (Gemini Integration)
- **Series Selection**: A Netflix-styled catalog where users select TV shows in target languages.
- **Dynamic Quizzes**: Users take multiple-choice quizzes uniquely generated by Gemini AI regarding the shows they watch.
- **Adaptive Flashcards**: If a user gets a question wrong, the app hits the backend to generate an adaptive Gemini-powered flashcard to reinforce their vocabulary.

### 4. Live Tutoring & Scheduling (Calendly)
- **Schedule Tab**: Users can browse active `instructor_sessions` mapped from the database, displaying language, level, mentor name, and time slots.
- **Booking Flow**: Uses `url_launcher` to kick the user to specific Calendly links.
- **My Schedule Widget**: On the main Dashboard, the app cross-references the User's ID with the `user_sessions` table to exclusively show classes the user has confirmed and booked.

## Architecture Blueprint
The database architecture uses linked UUIDs representing relationships between users, the content they consume (courses), the exams they take (quizzes/questions), their performance (flashcards generation), and their live coaching classes (sessions).
